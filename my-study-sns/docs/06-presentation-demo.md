#### 슬라이드 3-4: 프로젝트 소개

**배경**
- 학생들의 손글씨 노트 디지털화 어려움
- 시간표 등록의 번거로움
- 학습 자료 공유 플랫폼 부재

**목적**
- AI를 활용한 학습 콘텐츠 자동화
- 지식 공유 커뮤니티 구축

**목표**
1. 노트 이미지 → 마크다운 자동 변환
2. 시간표 이미지 → 일정 자동 등록
3. 게시글 → 학습 카드 자동 생성

#### 슬라이드 5-6: 문제 정의

**기존 서비스 한계**

| 서비스 | 한계 |
|--------|------|
| Google Keep | OCR 기능 없음 |
| 네이버 OCR | 단순 텍스트만, 수식 미지원 |
| 일반 SNS | 학습 목적 특화 X |

**My Study SNS 해결책**

| 문제 | 해결책 |
|------|--------|
| 손글씨 인식 | AI OCR + 2단계 교정 |
| 수식 지원 | LaTeX 자동 변환 |
| 시간표 등록 | 이미지 자동 분석 |
| 학습 정리 | AI 지식카드 생성 |

#### 슬라이드 7-9: 기술 스택

**프론트엔드**
- Next.js 15 (React 18)

**백엔드**
- Next.js API Routes

#### 슬라이드 10-12: 시스템 설계

**아키텍처 다이어그램**
```
클라이언트 → Next.js → API Routes → Supabase/Gemini
```

**데이터베이스 ERD**
- profiles, posts, knowledge_cards, hashtags...

**컴포넌트 구조**
- Pages, Components, Contexts, API Routes

#### 슬라이드 13-17: 주요 기능

**1. AI 스터디 노트 분석**
- 손글씨/인쇄물 자동 인식
- 2단계 OCR (추출 + 교정)
- 다중 이미지 페이지 병합
- LaTeX 수식 자동 변환

**2. 시간표 이미지 분석**
- 사진 한 장으로 시간표 등록
- 24시간제/12시간제/교시제 자동 감지
- 충돌 검사 및 편집 기능

**3. AI 지식카드 생성**
- 게시글 → 학습 카드 자동 변환
- 카테고리, 키워드 자동 분류
- 카로셀 형태로 표시

**4. 학습 커뮤니티**
- 게시판 (자유, Q&A, 스터디노트)
- 해시태그 기반 탐색
- 마크다운 + 수식 렌더링

## 6.3 예상 질문 및 답변

### 기술 관련 질문

**Q1: 왜 Gemini API를 선택했나요?**
```
A: Gemini는 멀티모달 분석이 뛰어나고, 이미지와 텍스트를 동시에 처리할 수 있습니다.
특히 손글씨 인식과 수식 변환에서 높은 정확도를 보여주었습니다.
또한 Gemini 2.5 Pro와 Flash 중 용도에 맞게 선택할 수 있어 비용 최적화가 가능합니다.
```

**Q2: 2단계 OCR의 장점은 무엇인가요?**
```
A: 첫 번째 장점은 정확도입니다. Phase 2에서 AI가 OCR 오류를 교정하여 최종 정확도를 높입니다.
두 번째는 효율성입니다. 신뢰도가 높으면 Phase 2를 스킵하여 불필요한 처리를 줄입니다.
세 번째는 투명성입니다. 교정 내역을 사용자에게 보여주어 신뢰를 높입니다.
```

**Q3: 데이터베이스 보안은 어떻게 처리했나요?**
```
A: Supabase의 Row Level Security(RLS) 정책을 적용했습니다.
사용자는 자신의 데이터만 수정/삭제할 수 있고, 읽기는 공개 게시글에 한해 허용됩니다.
또한 모든 API에서 인증 확인 후 처리합니다.
```

### 프로젝트 관련 질문

**Q4: 가장 어려웠던 점은 무엇인가요?**
```
A: 다양한 형태의 손글씨와 시간표를 인식하는 것이 가장 어려웠습니다.
특히 시간표의 12시간제/24시간제/교시제를 자동으로 감지하는 로직을 설계하는 데 많은 시행착오가 있었습니다.
```

**Q5: 향후 개선 계획은 무엇인가요?**
```
A: 첫째, 모바일 앱 개발로 접근성을 높이려 합니다.
둘째, 이미지 병렬 처리로 분석 속도를 개선하려 합니다.
셋째, 사용자 학습 패턴 분석을 통한 맞춤 추천 기능을 추가하려 합니다.
```
